# OBSERVATION_CHECKLIST_V11

## Day 0

- verify previously submitted URLs are already visible to Google tools where applicable
- save one baseline Search Console screenshot or note set
- do not change the site

## Day 1-3

- check Search Console impressions
- check whether `/about/`, `/topics/`, and `/distinctions/` begin appearing as known URLs
- check whether branded queries still resolve mostly to LinkedIn or begin sharing space with the site
- do not add new pages

## Day 4-7

- inspect query drift
- inspect whether Google starts associating corpus terms with `ivankotov.eu` more often
- note if LinkedIn still dominates all branded queries

## Day 7-10

- decide whether the site needs `no change`, `one precision pass`, or `one cleanup pass`
- use the decision matrix rather than instinct

## What not to do

- do not add new public pages during observation
- do not keep resubmitting the same URLs without material change
- do not overread one day of weak data
- do not panic because one query still misfires

## What counts as meaningful change

- a previously unseen page starts getting impressions
- a branded or topic query begins surfacing a site page instead of only external anchors
- Google-selected canonical for a key page becomes stable and correct
- `/topics/`, `/distinctions/`, or `/about/` starts appearing directly in query results

## What counts as premature panic

- one or two days with flat impressions
- branded queries still being dominated by LinkedIn early in the window
- a topic page not appearing immediately after publication
- low-volume Search Console data that has not yet stabilized
