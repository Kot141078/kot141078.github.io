# Live Shape Summary

## Home role

The home page is a public identity-and-navigation surface, not a landing page. It introduces Ivan Kotov as an AI Systems Architect, frames AGI as Advanced Global Intelligence, and routes readers into identity, corpus structure, practical pages, topic pages, canonical entry nodes, publications, releases, evidence, and the book layer.

## Identity role

Identity is distributed across:

- `/about/` as the short public profile
- `/corpus-map/` as the corpus-role map
- `/publications/` as the public-works surface

This is coherent, but these pages partially overlap in "who / what is this corpus" work.

## Source / download role

Source and download roles are clear:

- `/library/` = source surfaces
- `/downloads/` = release and file surfaces
- `/publications/` = public works
- `/evidence/` = reviewable contour
- `/releases/` = release timeline

This is strong for auditability, but it creates several adjacent inventory-style pages.

## Practical role

The practical layer is well-formed:

- `/services/` presents three review directions
- `/use-cases/` maps six practical situations
- `/contact/` stays intentionally minimal and anchor-based

This practical block reads as serious and non-salesy. It does not appear visually or semantically broken.

## Topic role

The topic layer is also coherent:

- `/topics/` acts as the hub
- `/ai-governance/`, `/local-first-ai/`, and `/long-lived-ai-entities/` provide external reading angles into the same corpus

The topics are concise and aligned with the canonical pages, but they add another entry mode on top of already existing identity/path/source/practical layers.

## Has the site become too wide?

Moderately, yes.

Not because of post-freeze drift, but because the current frozen shape already exposes many neighboring entry surfaces at once. The site is still usable, but its public shape is broader than the minimum needed for first orientation.

## Has navigation drifted?

No major drift is visible.

The navigation remains internally coherent and the live site matches the repo at a high level. The main risk is density, not inconsistency.

## Overlap / redundancy

The main overlaps are:

- `About` vs `Corpus map` vs `Start here` vs `Reading path`
- `Library` vs `Downloads` vs `Publications` vs `Evidence` vs `Releases`
- `Services` vs `Use cases` vs `Topics`

These overlaps are not severe enough to justify a cleanup-first decision while AGL and Volume II remain absent.

## Pass implication

The next pass should be an integration pass.

Rationale:

- the frozen public surface is stable
- the visual layer is acceptable
- the main missing work is structural integration of AGL and Volume II into the already existing corpus/navigation system
