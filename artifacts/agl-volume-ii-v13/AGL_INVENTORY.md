# AGL Inventory

## Canonical title

Actor Grounding Layer (AGL) v0.1

## Short definition

Bounded upstream grounding layer for signals, actors, nodes, and perceptual paths before runtime reliance or action progression is allowed.

## Short role in ecosystem

AGL sits upstream of ordinary progression and review. It determines whether a source is grounded enough in real present execution state to support runtime reliance, initiation, escalation, or review at all.

## Relation to adjacent layers

- L4:
  - L4 constrains physical viability and feasibility.
  - AGL constrains source grounding and runtime reliance.
- SER:
  - AGL sits before ARL-style review and continuity/governance progression.
  - It prevents procedural conflict logic from starting on already abstracted or degraded initiation.
- ARL:
  - Confirmed.
  - Source set states that AGL sits upstream of ARL procedural dispute review and does not replace it.
- L4 Witness / hardware perimeter:
  - Confirmed.
  - Source set states that AGL does not replace L4 Witness or L4 Hardware Perimeter.
  - Hardware Perimeter and AGL are connected as an explicit bridge: `L4 Hardware Perimeter ↔ Actor Grounding Layer ↔ ARL / runtime reliance`.

## What AGL is not

- Not a replacement for ARL procedural dispute review.
- Not a replacement for L4 Witness evidence traceability.
- Not a replacement for L4 Hardware Perimeter.
- Not the full ECC implementation bridge.
- Not a second sovereign center above the stack.
